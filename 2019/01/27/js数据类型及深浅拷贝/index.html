js数据类型及深浅拷贝

发表于 2019-01-27 | 分类于 js |
字数统计: 字 | 阅读时长 ≈ 分钟

栈（stack）和堆（heap）

stack为自动分配的内存空间，由系统自动释放；而heap则是动态分配的内存，大小不固定。

javascript 数据类型

js数据类型分为两种：基本数据类型和引用数据类型

六种基本数据类型（值类型）
  1. Number
  2. String
  3. Boolean
  4. Undefined
  5. Null
  6. Symbol(ES6)
    引用数据类型
    object
    JS中除基本数据类型以外都是对象，数组是对象，函数是对象，正则表达式是对象
    js中基本上数据类型存放在栈中的，它们数据大小确定，内存空间大小可以分配，是直接按值存放的，所以可以直接按值访问。
    引用数据类型是存放在堆内的对象，变量其实是保存在栈内存中的一个指针，这个指针指向堆内存地址。通过这个地址可以快速查找到保存在堆内存中的对象。

    深浅拷贝

    深拷贝是值拷贝，修改拷贝的值对原来存储的值不影响，浅拷贝拷贝的是指向某个堆内存空间的指针，所以修改值会对原值产生影响。
    浅拷贝问题踩坑
    Object.assign({},object)
    本以为使用Object.assign进行对象的合并拷贝应该不会出现问题，使用vue开发组件过程中一直出现拷贝后对象出现于原值出现绑定的现象。查了下发现Object.assign()方法实行的是浅拷贝，举个栗子。
    如果对象的值是基本数据类型的，拷贝属性值是没有任何问题的。如果是引用数据类型浅拷贝实现的是指针的拷贝，所以出现修改拷贝值对原始值产生影响的问题。
    深拷贝
  7. JSON.parse(JSON.stringify(obj))
    但由于JSON的局限性，该方法也不是万能的，比如对象属性是undefined、函数、symbol或xml对象，该属性会被JSON.stringify()过滤掉，导致拷贝时缺少属性。
